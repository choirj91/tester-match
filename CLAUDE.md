## 프로젝트 목표

Google Play Closed Testing 12명/14일 요건을 해소하는 한국형 품앗이 매칭 플랫폼의 **12주 MVP 출시**.

대상: 한국 인디 안드로이드 개발자 (v1) → 영어권 글로벌 (v2)

## 5레이어 구조 (절대 규칙)

```
01-source/    원천 — 읽기 전용 (절대 수정 X)
02-reference/ 참조 — 추가만, 기존 변경 X
03-output/    산출물 — 여기만 자유롭게 생성/수정
04-review/    품질관리 — 누적 (이력은 덮어쓰기 X)
05-harness/   엔진 — 점진적 개선
```

- `01-source/`는 **절대 수정하지 않는다**. 기획·BP·ADR이 흔들리면 모든 결정이 무너진다.
- 모든 코드·정책 산출물은 `03-output/`에만 둔다.
- 세션 결정·심사 결과는 `04-review/history/`에 누적한다.

## 작성 7원칙

> 모든 산출물(코드·정책·문서) 작성 시 준수.

1. **원천 분리**: 01-source/ 데이터는 인용·복제만, 수정 X
2. **수치화**: "빠른", "안정적인" 금지 → ms·% 단위로
3. **비교 데이터**: 모든 기술 주장에 Before/After 또는 경쟁사 비교
4. **예시 제시**: 매칭·결제·체크인 워크플로우에 구체적 시나리오
5. **결정 추적**: 새 결정은 ADR 추가, 기존 결정 변경은 ADR 갱신
6. **무료 우선**: 기능 추가 시 무료 티어 영향 분석 (08_tech_stack §2 표 참조)
7. **정책 우선**: Google Play incentivized 정책 위반 가능 기능은 자동 차단 (F-RVW-* 보류)

## 작업 원칙

- 파일을 **읽기 전에 수정하지 않는다**.
- 간결하게 출력하되 추론은 꼼꼼하게.
- 기존 파일 편집을 우선한다. 불필요한 새 파일을 만들지 않는다.
- 확실하지 않으면 솔직히 말한다. 파일 경로를 추측하지 않는다.
- **결제·정책·약관 변경은 사용자 승인 필수**.
- 환불·결제 관련 코드는 멱등성(idempotent) 검증 필수.
- 14일 카운트 로직은 timezone-safe 단위 테스트 필수.

## 절대 금지

- `01-source/` 직접 수정
- 잠금 결정과 충돌하는 코드 작성 (ADR 갱신 없이)
- Google Play 리뷰·별점 인센티브 기능 활성화 (F-RVW-* 보류)
- 결제 정보(카드번호·CVC) DB 저장
- 개인정보 마스킹 없는 로그 전송
- `--no-verify`, `--force` 등 안전장치 우회
