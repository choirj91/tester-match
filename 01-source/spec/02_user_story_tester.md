# 유저 스토리 — 테스터 (Tester)

> Persona: **다른 개발자의 앱을 14일간 테스트해주고 크레딧을 적립받는 사용자**
> 작성일: 2026-05-04 / Version: v0.1

---

## ⚠️ Critical Policy Notice

| 항목 | 위험 | 본 문서 반영 |
|---|---|---|
| US-TST-09 (Google Play 별점·리뷰 제출) | **Google Play 정책 위반 (incentivized reviews)** | "Phase 1 보류" 표기 |
| US-TST-10 (포인트 외부 전환) | 선불전자지급수단 발행업 등록 + 제휴 6~12개월 | v3 활성화 |

## 0. Persona 상세

### Persona A — "용돈벌이형" 사용자
| 항목 | 내용 |
|---|---|
| 이름(가상) | 박학생 (24세, 대학생) |
| 동기 | 짧은 시간에 부담 없이 용돈 벌고 싶음 (월 1~3만원) |
| 디바이스 | 갤럭시/픽셀 등 안드로이드 1대 (메인 폰) |
| 활동 시간 | 주로 저녁 9~12시 |
| 우려 | 가짜 앱·악성앱 설치 우려, 시간 낭비 우려 |
| 필요 | 신뢰할 수 있는 매칭, 빠른 정산, 14일 알림 |

### Persona B — "동료 개발자형" 사용자
| 항목 | 내용 |
|---|---|
| 이름(가상) | 이개발 (32세, 인디 개발자) |
| 동기 | 내 앱도 12명 필요해서 품앗이 차원에서 참여 |
| 디바이스 | 안드로이드 + 에뮬레이터 (Google 정책상 실기기 우선) |
| 활동 시간 | 주말·저녁 |
| 우려 | 14일을 매일 챙기는 게 번거로움 |
| 필요 | 자동 알림, 한 번에 여러 앱 동시 테스트 가능 |

## 1. 가입·온보딩

### US-TST-01: 가입 및 첫 매칭 안내

> **As a** 처음 방문한 잠재 테스터  
> **I want to** 가입 직후 어떻게 크레딧을 모으는지 한 화면으로 이해하고 싶다  
> **So that** 곧바로 첫 매칭에 참여해 적립을 시작할 수 있다

**Acceptance Criteria**
- [ ] Google OAuth 가입 30초 이내
- [ ] 온보딩 3단계: (1) 작동 원리 (2) 14일 약속 (3) 첫 매칭 시작
- [ ] 약관 동의 시 "14일 약속" 별도 체크박스 (강제 옵트아웃은 페널티 안내)
- [ ] 가입 즉시 환영 크레딧 200원 분량 (1회성)

---

### US-TST-02: 디바이스 등록

> **As a** 새 테스터  
> **I want to** 내 안드로이드 디바이스 정보(모델, OS 버전)를 등록해두고 싶다  
> **So that** 앱이 요구하는 디바이스 조건과 매칭이 정확히 이루어진다

**Acceptance Criteria**
- [ ] Google Play의 SafetyNet/Play Integrity 결과를 기반으로 실기기 검증 (선택)
- [ ] 디바이스 정보는 매칭 알고리즘에만 사용 (제3자 비공개)
- [ ] 1계정 = 최대 2 디바이스 (메인 + 보조)
- [ ] 동일 디바이스를 여러 계정에 등록 시 자동 차단 (fake account 방지)

---

## 2. 매칭

### US-TST-03: 매칭 받기

> **As a** 활성 테스터  
> **I want to** 내 조건(타임존, 카테고리)에 맞는 앱을 자동으로 추천받고 싶다  
> **So that** 직접 검색하지 않아도 적합한 앱을 빠르게 시작할 수 있다

**Acceptance Criteria**
- [ ] 메인 화면에 "지금 시작 가능한 앱" 카드 리스트 (최대 10개)
- [ ] 각 카드: 앱 이름, 카테고리, 보상 크레딧, 14일 종료 예정일, "참여" 버튼
- [ ] 카드 클릭 시 상세: 앱 설명, 스크린샷, 개발자 닉네임, Trust Score
- [ ] "참여" 클릭 → 즉시 옵트인 처리 + Google Play 초대 링크 표시 + 이메일 발송

---

### US-TST-04: 급구 채널 (Boost) 우선 노출

> **As a** 적립을 빠르게 하고 싶은 테스터  
> **I want to** 결제된 급구 앱을 우선 추천받고 싶다  
> **So that** 보상이 더 큰 앱에 먼저 참여할 수 있다

**Acceptance Criteria**
- [ ] 급구 채널 별도 탭 + 푸시 알림 옵션
- [ ] 급구 앱은 일반 풀 대비 2배 크레딧 (예: 1,000 → 2,000)
- [ ] 선착순 마감 (12명 도달 시 자동 종료)
- [ ] 급구 알림 ON 사용자에게는 새 등록 시 즉시 푸시

---

## 3. 14일 유지·체크인

### US-TST-05: 자동 체크인 알림

> **As a** 옵트인한 테스터  
> **I want to** 매일 한 번 체크인 알림을 받아 기억하지 않아도 되게 하고 싶다  
> **So that** 14일 카운트가 끊기지 않고 적립까지 무사히 도착할 수 있다

**Acceptance Criteria**
- [ ] 알림 시각은 사용자 선택 (오전/저녁) — 기본은 저녁 8시 (현지 타임존)
- [ ] 알림 채널: 이메일 (필수), 카카오 알림톡(v2), 푸시(v2)
- [ ] 알림 본문에는 "Google Play에서 5초 실행" 가이드 포함
- [ ] 14일 중 N일차 진행률 표시
- [ ] 5일 연속 미체크인 시 자동 페널티 경고 (Trust Score -5)

---

### US-TST-06: 체크인 인증

> **As a** 체크인 알림을 받은 테스터  
> **I want to** 1탭으로 "오늘 사용 완료" 인증을 끝내고 싶다  
> **So that** 매일 부담 없이 14일을 채울 수 있다

**Acceptance Criteria**
- [ ] 기본 인증: 1탭 ("Yes, I used it today") — 자가 보고
- [ ] 무작위 검증: 14일 중 2~3회 무작위로 "스크린샷 첨부 요청"
- [ ] 스크린샷은 앱의 임의 화면 + 시스템 시계 포함 (위조 방지)
- [ ] 이미지 업로드는 모바일 친화적 (드래그·카메라 직접 캡처)
- [ ] 부정 행위 의심 시 운영팀 검토 큐로 이동

---

### US-TST-07: 옵트아웃 처리

> **As a** 테스트 중 부득이 중단해야 하는 테스터  
> **I want to** 명시적으로 옵트아웃하고 싶다  
> **So that** 무책임하게 사라지지 않고 페널티를 최소화할 수 있다

**Acceptance Criteria**
- [ ] 옵트아웃 사유 입력 필수 (앱 문제 / 개인 사정 / 기타)
- [ ] 자발적 옵트아웃: Trust Score -10 (페널티)
- [ ] 앱 자체에 문제(크래시, 악성 의심) 사유 옵트아웃: 페널티 면제 + 운영팀 알림
- [ ] 14일 중 7일차 이후 옵트아웃: 부분 크레딧 지급 (50%)

---

## 4. 크레딧 적립·관리

### US-TST-08: 적립 및 잔액 확인

> **As a** 14일을 완주한 테스터  
> **I want to** 즉시 크레딧을 받고 잔액을 확인할 수 있다  
> **So that** 다음 매칭에 바로 사용할 수 있다

**Acceptance Criteria**
- [ ] 14일 완주 → 즉시 크레딧 자동 적립 (지연 없음)
- [ ] 적립 비율: **1매칭 = 1,000 크레딧** (개발자가 1매칭당 1,000원 지불)
  - 이 중 200 크레딧은 플랫폼 수수료, **테스터에게는 800 크레딧 지급**
- [ ] 급구 매칭은 2배 (1,600 크레딧)
- [ ] 적립 내역 무한 스크롤, 필터(완주/페널티/충전), CSV 내보내기

---

### US-TST-09: 크레딧 사용

> **As a** 크레딧을 모은 테스터  
> **I want to** 내 앱 등록에 크레딧을 사용할 수 있다  
> **So that** 별도 결제 없이 12명 매칭을 진행할 수 있다

**Acceptance Criteria**
- [ ] 크레딧 사용처(v1): 내 앱 등록·매칭 비용 차감
- [ ] 부족분은 KRW로 추가 충전 (1원 = 1 크레딧)
- [ ] 사용 시 즉시 잔액 차감, 환불 정책에 따라 환불 가능

---

## 5. 신뢰·평판

### US-TST-10: Trust Score 확인

> **As a** 테스터  
> **I want to** 내 Trust Score와 점수 산정 근거를 확인할 수 있다  
> **So that** 점수를 높여 더 좋은 매칭을 받을 수 있다

**Acceptance Criteria**
- [ ] 점수 범위: 0~100 (시작은 50)
- [ ] 가산 항목: 14일 완주(+5), 무작위 검증 통과(+1), 신고 누적 0회 유지
- [ ] 감산 항목: 자발적 옵트아웃(-10), 미체크인(-5), 신고 접수(-15), 부정 행위(-30)
- [ ] 70점 이상 → 급구 매칭 우선권
- [ ] 30점 미만 → 신규 매칭 일시 정지 (회복 미션 안내)

## 6. ⚠️ 위험 영역 (Phase 1 보류 권장)

### US-TST-11: Google Play 별점·리뷰 작성 — Critical Policy Risk

> **As a** 추가 적립을 원하는 테스터  
> **I want to** 14일 완주 후 별점·리뷰 작성 미션을 받아 추가 크레딧을 적립할 수 있다  
> **So that** 시간당 적립률을 높일 수 있다

**❌ Critical Policy Risk**

본 스토리는 사용자께서 명시적으로 요청하신 항목이나, 다음 정책에 직접 위반됨:
- [Google Play "Incentivized ratings, reviews and installs" 정책](https://android-developers.googleblog.com/2017/06/google-plays-policy-on-incentivized.html)
- 적발 시 **테스터의 Google 계정도 정지 가능** (테스터에게도 직접적 손해)

**권장 대안 (US-TST-11a)**
- "비공개 테스트 중 피드백/버그 제보"에 한해 크레딧 지급
- 공개 리뷰는 자발적 안내만 (인센티브 없음)

**Phase 1 결정**: v1 비활성, 출시 전 결정 필요

---

### US-TST-12: 외부 포인트 전환 — v3 활성화

> **As a** 적립 크레딧을 다른 곳에서도 쓰고 싶은 테스터  
> **I want to** 네이버페이/카카오페이 포인트로 전환할 수 있다  
> **So that** 크레딧을 실생활 결제에 사용할 수 있다

**Acceptance Criteria (v3)**
- [ ] 전환 비율: 1,000 크레딧 = 800 네이버페이 포인트 (수수료 20%)
- [ ] 전환 단위: 5,000 크레딧 이상부터 (소액 다발 송금 방지)
- [ ] 월 전환 한도: 50,000 크레딧 (자금세탁방지)
- [ ] 전환 신청 → 익영업일 처리 (수기 검증)

**Phase 1 결정**
- v1: UI 메뉴는 노출하되 "Coming Soon" 비활성
- v2: 단일 제휴처(예: 네이버페이) 협상 시작
- v3: 발행업 등록 완료 후 활성화
