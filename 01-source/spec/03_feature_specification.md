## 0. 본 문서의 목적

이 문서는 v1 MVP에서 **무엇을 만들고, 무엇을 만들지 않을지**를 명확히 정의한다. 모든 기능은 다음 4가지 속성을 갖는다:

- **ID**: `F-XXX-YYY` (도메인-번호)
- **우선순위**: P0(필수) / P1(중요) / P2(있으면 좋음) / P3(보류)
- **수용 기준 (Acceptance Criteria)**
- **연결되는 유저 스토리** (US-DEV-XX / US-TST-XX)

```

       + (테스터 활성도 × 0.2)
       + (카테고리 일치 보너스 × 0.1)
```

## 4. 비기능 요구사항 (Non-Functional Requirements)

| 분류 | 요구사항 |
|---|---|
| **성능** | 페이지 LCP ≤ 2.5s, 매칭 알고리즘 응답 ≤ 500ms |
| **가용성** | 99.5% (월 다운타임 ≤ 3.6시간) |
| **보안** | OWASP Top 10 대응, 결제 정보 미저장, 토큰 단방향 해시 |
| **개인정보** | PIPA(한국)/GDPR 대응, 최소 수집, 7일 내 삭제 처리 |
| **접근성** | WCAG 2.1 AA 일부 준수 (키보드 탐색, 색상 대비) |
| **모바일** | 반응형 (모바일 우선 디자인), iOS Safari/Android Chrome 최신 2버전 지원 |
| **로깅** | Sentry로 모든 에러 자동 캡처, PII 마스킹 |
| **백업** | DB 일 1회 자동 백업, 30일 보관 |
| **i18n** | v1: 한국어만 / v2: 영어 추가 (i18n 키 구조는 v1부터 적용) |

## 6. 결정 대기 항목 (Open Decisions)

- [ ] 도메인 확정 (testermatch.com / cotest.dev / etc.)
- [ ] Boost 가격 확정 (USD $29 vs KRW 39,000)
- [ ] 크레딧 적립 비율 (개발자→테스터 80%? 70%?)
- [ ] Trust Score 가중치 세부값 (베타 후 튜닝)
- [ ] 무작위 스크린샷 검증 빈도 (14일 중 2회 vs 3회)
- [ ] 환불 정책 한도 (충전 후 7일 vs 14일)
- [ ] 리뷰 캠페인(F-RVW-*) 활성화 여부 — **법무 검토 후**
