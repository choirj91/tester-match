## 1. 회원가입 / 온보딩

### US-DEV-01: 빠른 회원가입

> **As a** 인디 개발자  
> **I want to** Google 계정으로 30초 안에 가입하고 싶다  
> **So that** 가입 절차에서 이탈하지 않고 바로 앱 등록까지 진행할 수 있다

**Acceptance Criteria**
- [ ] Google OAuth 버튼 클릭 → 권한 허용 → 가입 완료까지 3 클릭 이내
- [ ] 가입 시 자동으로 타임존(브라우저) 감지 → 프로필에 저장
- [ ] 가입 직후 "환영 + 첫 앱 등록" CTA가 노출됨
- [ ] 이메일 인증은 Google OAuth로 대체 (별도 인증 메일 없음)
- [ ] 가입 즉시 환영 크레딧 1,200원 분량 지급 (= 첫 매칭 1건 시도 가능)

**Out of Scope (v1)**
- 카카오/네이버 OAuth (v2)
- 휴대폰 본인인증 (v2 — 한국 시장 확장 시)

> **As a** 가입 직후 개발자  
> **I want to** 내 활동 시간대·주력 카테고리를 설정해두고 싶다  
> **So that** 매칭 시 비슷한 시간대의 테스터를 우선 배정받을 수 있다

**Acceptance Criteria**
- [ ] 프로필 항목: 닉네임, 타임존, 관심 카테고리(다중 선택), 디바이스 정보
- [ ] 모든 항목은 선택사항 (필수 아님) — 단, 매칭 정확도 안내 노출
- [ ] 프로필 수정 후 즉시 저장 (별도 저장 버튼 없음, blur 시 자동 저장)

## 2. 앱 등록

### US-DEV-03: 앱 등록 폼

> **As a** 개발자  
> **I want to** 내 앱 정보와 Google Play 비공개 테스트 초대 링크를 등록하고 싶다  
> **So that** 매칭된 테스터가 자동으로 초대 링크를 받을 수 있다

**Acceptance Criteria**
- [ ] 입력 항목: 앱 이름, 카테고리, 한 줄 설명(140자), Google Play 초대 링크, 필요 테스터 수(기본 12)
- [ ] Google Play 초대 링크는 정규식 검증 (`play.google.com/apps/test/...`)
- [ ] 초대 링크가 유효한지 시스템이 직접 HEAD 요청으로 1차 검증
- [ ] 등록 시 필요 크레딧 차감 안내 (12명 × 매칭당 1,000크레딧 = 12,000크레딧)
- [ ] 크레딧 부족 시 "충전" 또는 "Boost로 한번에" CTA 분기 노출

---

### US-DEV-04: 앱 수정·일시정지·삭제

> **As a** 등록한 개발자  
> **I want to** 앱 정보를 수정하거나 매칭을 일시 정지할 수 있다  
> **So that** 초대 링크가 만료되거나 출시 일정이 변경될 때 유연하게 대응할 수 있다

**Acceptance Criteria**
- [ ] 매칭 진행 중인 앱은 "수정" 시 신규 매칭만 신규 정보로 적용 (이미 매칭된 테스터는 영향 없음)
- [ ] "일시정지"는 신규 매칭만 멈춤 (진행 중인 14일 카운트는 유지)
- [ ] "삭제"는 매칭 0건 상태에서만 가능 (진행 중일 때는 일시정지만 가능)
- [ ] 삭제 시 미사용 크레딧은 즉시 환원

## 3. 매칭

### US-DEV-05: 일반 매칭 (Regular Pool)

> **As a** 시간 여유가 있는 개발자  
> **I want to** 크레딧만 차감하고 일반 풀에서 자동 매칭받고 싶다  
> **So that** 추가 결제 없이 다른 개발자의 앱을 테스트해주고 받은 크레딧으로 내 앱을 등록할 수 있다

**Acceptance Criteria**
- [ ] 일반 풀 등록 시 즉시 큐에 진입 (FIFO + Trust Score 가중)
- [ ] 매칭 진행률을 대시보드에서 실시간 확인 (12명 중 N명 매칭)
- [ ] 매칭된 테스터별로 "옵트인 시각 / 14일 남은 일수" 표기
- [ ] 매칭 실패(72시간 내 0명)시 자동 알림 + 사유 안내

---

### US-DEV-06: 급구 (Boost) 결제

> **As a** 출시가 임박한 개발자  
> **I want to** 결제로 즉시 12명 매칭을 보장받고 싶다  
> **So that** 14일 후 정확히 프로덕션 액세스를 신청할 수 있다

**Acceptance Criteria**
- [ ] Boost 등록 시 가격 옵션 노출: 12명/24h(49,000원) / 12명/48h(29,000원) — 가격은 Open Question
- [ ] 결제 수단 (v1): 토스페이먼츠 (국내카드/계좌이체/카카오페이/네이버페이/토스결제)
- [ ] 결제 수단 (v2): Stripe Checkout (해외카드/Apple Pay/Google Pay) — 영어권 확장 시 추가
- [ ] 결제 직후 "급구" 채널 최상단 노출 + 푸시 알림 발송
- [ ] **SLA 미달 시 자동 환불** (예: 48h 약속, 36h 시점에 8명만 매칭 → 4명 부족 비례 환불)
- [ ] 환불 사유는 결제 영수증과 별도 메일로 발송
- [ ] 환불 처리 시간 명시 (토스 카드: 3~5영업일, 토스 계좌이체/간편결제: 1~3영업일)

---

### US-DEV-07: 매칭 보충

> **As a** 매칭 진행 중인 개발자  
> **I want to** 테스터가 14일 미만에 옵트아웃하면 자동으로 보충받고 싶다  
> **So that** 14일 카운트가 리셋되지 않도록 즉시 새 테스터가 들어온다

**Acceptance Criteria**
- [ ] 옵트아웃 감지 시 즉시 큐 최상단에 보충 매칭 등록
- [ ] 보충 매칭이 24시간 내 미완료 시 개발자에게 경고 알림
- [ ] 보충은 무료 (Boost 결제 시 SLA에 포함)

## 4. 크레딧 관리

### US-DEV-08: 크레딧 충전·잔액 확인

> **As a** 개발자  
> **I want to** 크레딧을 KRW로 충전하고 잔액·내역을 확인할 수 있다  
> **So that** 일반 풀에서 안정적으로 매칭을 진행할 수 있다

**Acceptance Criteria**
- [ ] 충전 비율: **1원 = 1 크레딧** (1,000원 충전 시 1,000 크레딧)
- [ ] 충전 단위: 1,000 / 5,000 / 10,000 / 30,000 / 50,000원 + 직접 입력
- [ ] 결제 수단: 토스페이먼츠(v1, 국내) / Stripe(v2, 해외 영어권 확장 시)
- [ ] 잔액 = (적립 + 충전) - (사용 + 만료)
- [ ] 내역은 무한 스크롤, 필터(적립/사용/충전/환불), CSV 내보내기 지원
- [ ] **외부 포인트 전환(네이버페이/카카오페이) 메뉴는 v1에서 "Coming Soon" 비활성 노출** (UI는 사전 설계, 실제 활성화는 v3)

**Out of Scope (v1)**
- 외부 포인트 실제 전환 (v3, 발행업 등록 후)
- 크레딧 양도/거래 (v3)
- 크레딧 만료 정책 — v1은 무기한, v2부터 발급 후 12개월

### US-DEV-09: 크레딧 환불

> **As a** 미사용 크레딧을 보유한 개발자  
> **I want to** 결제 후 7일 내 미사용 크레딧을 전액 환불받을 수 있다  
> **So that** 안심하고 첫 충전을 할 수 있다

**Acceptance Criteria**
- [ ] 충전 후 7일 이내 + 사용 0건 → 100% 환불
- [ ] 일부 사용 시 미사용분만 비례 환불 (단 결제수수료 차감)
- [ ] 환불 신청은 마이페이지 → 자동 승인 (CS 개입 없음)

## 6. ⚠️ 위험 영역 (Phase 1 보류 권장)

### US-DEV-12: Google Play 리뷰 캠페인 — Critical Policy Risk

> **As a** 출시 후 별점이 필요한 개발자  
> **I want to** Tester Match를 통해 Google Play 별점·리뷰 작성을 의뢰할 수 있다  
> **So that** 초기 별점·리뷰 수를 빠르게 확보할 수 있다

**❌ Critical Policy Risk**

본 스토리는 사용자께서 명시적으로 요청하신 항목이나, 다음 정책에 직접 위반됨:
- [Google Play "Incentivized ratings, reviews and installs" 정책](https://android-developers.googleblog.com/2017/06/google-plays-policy-on-incentivized.html)
- 적발 시 (1) 우리 플랫폼 이용 모든 개발자의 앱 정지, (2) 토스페이먼츠/Stripe 가맹점 차단, (3) 개발자 계정 영구 정지 가능

**권장 대안 (US-DEV-12a)**
- "비공개 테스트 중 피드백/버그 제보"에만 크레딧 지급 — 공개 리뷰는 인센티브 없이 자발적 안내만
- 출시 후 자발적 리뷰는 "리뷰 요청 인앱 다이얼로그" 가이드 제공 (Google 공식 In-App Review API 사용)

**Phase 1 결정**
- v1 코드에서 본 기능 **비활성** (메뉴/DB만 정의, 실제 동작 차단)
- 출시 전 법무 검토 + 사업적 위험 감수 결정 필요
- 활성화 시 **별도 회사/브랜드 분리** 검토 권장 (메인 브랜드 보호)

**Acceptance Criteria (만약 활성화 시)**
- [ ] 본 기능은 별도 약관 동의 필요 ("정책 위반 가능성을 인지함")
- [ ] 결제 페이지에 명시적 경고 노출
- [ ] 우리 측 면책 조항 명시
- [ ] 모든 거래 내역은 별도 DB에 격리 보관
