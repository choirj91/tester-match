# 사용자 여정도 (User Flow) — Tester Match v1

> 작성일: 2026-05-04 / Version: v0.1
> 본 문서는 **핵심 사용자 흐름**을 Mermaid 다이어그램으로 정의한다.
> 노션 import 시 Mermaid 블록은 자동 렌더링됨.

---

## 1. 신규 가입 → 첫 매칭까지 (개발자)

```mermaid
flowchart TD
    A[랜딩 페이지 방문] --> B{회원가입 결정?}
    B -->|Yes| C[Google OAuth 로그인]
    B -->|No| Z1[이탈]
    C --> D[약관 동의 + 14일 약속]
    D --> E[가입 완료 + 환영 크레딧 1,200원 분량 지급]
    E --> F[온보딩 3단계 안내]
    F --> G{선택}
    G -->|내 앱 등록| H[앱 등록 폼]
    G -->|먼저 테스트해보기| I[참여 가능 앱 둘러보기]
    H --> J[앱 정보 입력 + Google Play 초대 링크]
    J --> K{초대 링크 검증}
    K -->|실패| J
    K -->|성공| L{크레딧 충분?}
    L -->|예 12,000+ 크레딧 보유| M[일반 풀 등록 → 매칭 대기]
    L -->|아니오| N{선택}
    N -->|충전| O[KRW 충전 → 크레딧 적립 → M]
    N -->|급구로 즉시| P[Boost 결제 → 큐 최상단]
    M --> Q[매칭 진행률 대시보드 모니터링]
    P --> Q
    Q --> R[12명 매칭 완료]
    R --> S[14일 카운트 시작]
```

## 2. 신규 가입 → 첫 적립까지 (테스터)

```mermaid
flowchart TD
    A[랜딩 페이지 방문] --> B[Google OAuth 가입]
    B --> C[온보딩 + 14일 약속 동의]
    C --> D[가입 환영 크레딧 200원 분량]
    D --> E[참여 가능 앱 카드 리스트]
    E --> F{앱 선택}
    F -->|일반 풀| G[참여 클릭 → 옵트인 처리]
    F -->|급구| H[참여 클릭 → 옵트인 + 2배 보상 안내]
    G --> I[Google Play 초대 링크 표시 + 이메일]
    H --> I
    I --> J[Play Store에서 앱 설치 + 첫 실행]
    J --> K[Day 1 체크인 완료]
    K --> L[매일 체크인 알림 수신]
    L --> M{14일 동안 유지?}
    M -->|매일 체크인| N[Day 14 완주]
    M -->|중도 옵트아웃| Y1[Trust Score -10]
    M -->|5일 연속 미체크인| Y2[페널티 + 자동 옵트아웃]
    N --> O[크레딧 800원 자동 적립]
    O --> P{잔액 사용처}
    P -->|내 앱 등록에 사용| Q[크레딧 차감]
    P -->|계속 적립만| R[잔액 유지]
    P -->|외부 전환| S[Coming Soon - v3]
```

## 3. 14일 체크인 일일 흐름

```mermaid
flowchart LR
    A[저녁 8시 알림 발송] --> B[이메일/푸시 수신]
    B --> C{사용자 행동}
    C -->|체크인 클릭| D[1탭 인증]
    C -->|무시| E{미체크인 N일차}
    D --> F{무작위 검증 대상?}
    F -->|아니오| G[체크인 완료 → Day N+1 카운트]
    F -->|예 14일 중 2-3회| H[스크린샷 업로드 요청]
    H --> I{업로드 완료?}
    I -->|예| J[운영팀 자동 검증 → G]
    I -->|24h 내 미업로드| K[페널티 경고]
    E -->|N=1~4| L[다음날 알림에 강조]
    E -->|N=5| M[자동 옵트아웃 + Trust -5]
```

## 4. 매칭 알고리즘 흐름

```mermaid
flowchart TD
    A[테스터가 /testing/discover 접근] --> B[큐에서 후보 앱 조회]
    B --> C[필터 적용 - 디바이스/카테고리/언어]
    C --> D[순위 계산]
    D --> E[큐 진입시각 가중치 0.5]
    D --> F[개발자 Trust Score 0.2]
    D --> G[테스터 활성도 0.2]
    D --> H[카테고리 일치 0.1]
    E --> I[정렬]
    F --> I
    G --> I
    H --> I
    I --> J[상위 10건 카드 노출]
    J --> K{테스터 참여 클릭}
    K --> L[Match 레코드 생성]
    L --> M[개발자에게 알림]
    L --> N[테스터에게 초대 링크 전달]
    M --> O[14일 카운트 시작]
    N --> O
```

## 5. Boost 결제 + SLA 흐름

```mermaid
flowchart TD
    A[개발자 Boost 옵션 선택] --> B{시간 옵션}
    B -->|24h 보장| C[가격 KRW 49,000]
    B -->|48h 보장| D[가격 KRW 29,000]
    C --> E[결제 수단 선택]
    D --> E
    E -->|v1 KRW| F[토스페이먼츠]
    E -.->|v2 USD| G[Stripe Checkout]
    F --> H[결제 완료 Webhook]
    G -.-> H
    H --> I[급구 큐 최상단 등록]
    I --> J[테스터에게 푸시 알림]
    J --> K{SLA 카운트다운}
    K -->|시간 내 12명 매칭| L[성공 → 14일 카운트]
    K -->|시간 초과 부분 매칭| M[부족분 비례 환불]
    K -->|시간 초과 0명| N[전액 환불]
    M --> O[환불 영수증 발송]
    N --> O
```

## 6. 신고 → Trust Score 흐름

```mermaid
flowchart TD
    A[개발자 매칭 이상 발견] --> B[테스터 신고 폼]
    B --> C{신고 사유}
    C -->|즉시 옵트아웃| D[자동 검증 + Trust -10]
    C -->|미설치| E[자동 검증 + Trust -15]
    C -->|부정 행위| F[운영팀 검토 큐]
    F --> G{운영팀 판단}
    G -->|확인됨| H[Trust -30 + 경고]
    G -->|반복| I[Trust -100 + 계정 정지]
    G -->|기각| J[신고자에게 안내]
    D --> K[누적 신고 3회?]
    E --> K
    H --> K
    K -->|Yes| I
    K -->|No| L[기록만 유지]
```

## 7. 환불 신청 흐름

```mermaid
flowchart TD
    A[크레딧 환불 신청 진입] --> B{충전 후 7일 이내?}
    B -->|아니오| C[환불 불가 안내]
    B -->|예| D{사용 내역?}
    D -->|0건| E[100% 자동 환불 승인]
    D -->|일부 사용| F[미사용분 비례 환불]
    F --> G[결제 수수료 차감 안내]
    G --> H{사용자 동의}
    H -->|예| I[자동 환불 처리]
    H -->|아니오| J[취소]
    E --> K[토스페이먼츠 환불 API 호출 v1 / Stripe v2]
    I --> K
    K --> L{환불 성공?}
    L -->|예| M[영수증 발송 + 잔액 차감]
    L -->|실패| N[운영팀 큐로 이동]
```

## 8. ⚠️ 보류 흐름 — 리뷰 캠페인 (Critical Risk)

```mermaid
flowchart TD
    A[개발자가 리뷰 캠페인 등록 시도] --> B{기능 활성화?}
    B -->|v1 비활성| C[안내 페이지: '정책 검토 중']
    B -->|v2+ 별도 결정 후 활성| D[추가 약관 동의]
    D --> E[KRW 결제 - 별도 가격]
    E --> F[테스터에게 미션 노출]
    F --> G{테스터 수락?}
    G -->|예| H[Google Play 별점·리뷰 작성]
    H --> I[리뷰 인증 스크린샷]
    I --> J[크레딧 지급]
    J --> K{Google 적발?}
    K -->|예| L[정책 위반 → 우리 플랫폼 + 양측 모두 정지 위험]
    K -->|아니오| M[정상 진행]
```

> ❌ 이 흐름은 v1에서 **비활성** 상태로 유지. 흐름 정의만 사전 보유.

## 9. 권한·세션 흐름 (보안)

```mermaid
flowchart TD
    A[페이지 진입] --> B{세션 쿠키?}
    B -->|없음| C{Public 페이지?}
    C -->|예| D[정상 노출]
    C -->|아니오| E[/login 리다이렉트]
    B -->|있음| F{토큰 유효?}
    F -->|만료| G[Refresh 시도]
    G -->|성공| H[새 토큰 저장 → 진입]
    G -->|실패| E
    F -->|유효| I{Admin 페이지?}
    I -->|아니오| H
    I -->|예| J{Admin 권한?}
    J -->|예| H
    J -->|아니오| K[403 Forbidden]
```
