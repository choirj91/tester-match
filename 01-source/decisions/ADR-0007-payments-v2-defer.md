## Context

기획상 v1 12주 안에 들어가도록 예정된 항목:

- F-PAY-01 토스페이먼츠 결제 통합 (P0)
- F-CRD-01 크레딧 충전 (P0)
- F-CRD-03 앱 등록 시 크레딧 차감 (P0)
- F-CRD-04 환불 (P0)

이를 활성화하려면:

| 사전 작업 | 비용·기간 |
|---|---|
| 사업자등록 (전자상거래) | 1~2주, 무료 |
| 통신판매업 신고 | 1주 |
| 토스페이먼츠 사업자 가입·심사 | 2~4주 |
| 변호사 검토 (이용약관·개인정보처리방침·환불) | 100~200만원, 2~4주 |
| 선불전자지급수단 분류 사전 자문 (금감원) | 1~2주 |

총 6~8주 + 100~200만원이 결제 시작 전에 선소요.

### 현 시점 데이터

- v1 베타 단계, 사용자 수 한 자리수
- 14일 완주율, MAU, 매칭 성공률 등 PMF 지표 미수립
- 결제 시작 시점에 시장 적합성 미확인 → 위 비용·시간 투자 회수 불확실

### 후보

| 안 | 장점 | 단점 |
|---|---|---|
| A. 기획대로 v1 에 결제 포함 | 매출 즉시. 정량 검증과 동시 진행. | 6~8주 + 200만원을 사용자 부족 상태에서 선소요. v1 출시 지연. |
| B. **v1 무료 운영 → PMF 검증 → v2 결제** | 출시 빨라짐. 무료 마찰 제거로 사용자 확보 ↑. PMF 지표 모인 후 결제 도입 의사결정. | 매출 0 기간 길어짐. 무료 의존 사용자가 결제 전환 거부할 위험. |
| C. v1 결제 미니멀(토스만, 약관 검토 생략) | 어느 정도 매출 + 빠름. | 약관 미검토는 분쟁 시 회사 불리. 정책 위반 위험. |

---

## Decision

**B. v1 무료 운영 → PMF 검증 후 v2 에서 결제 활성화** 채택.

### v1 (현재) 운영 모델

- 모든 매칭·앱 등록은 **무료**
- 14일 완주 시 800 크레딧 적립은 **표시·기록만 유지** (v2 결제 활성 시 사용처 생김)
- 회원가입 보너스 등 적립 정책도 표시·기록 유지
- Boost(급구) UI 는 "준비중" 상태 유지, 결제 진입점 비활성

### v2 진입 트리거 (다음 중 하나)

| 지표 | 임계 |
|---|---|
| MAU | 300 이상 |
| 14일 완주율 | 60% 이상 |
| 매칭 성공률 (12명 충원) | 70% 이상 |
| 누적 등록 앱 수 | 100건 이상 |
| 크레딧 누적 발행량 | 100만 이상 (적립 기준) |

지표 도달 시 결제 시스템 도입 검토 시작.

### v2 도입 시 처리

- 누적 적립된 크레딧은 그대로 유지 (사용처 활성)
- 신규 충전 + 환불 + 토스 결제 동시 활성
- 약관·정책 v1.0 변호사 검토 후 시행
- 사업자등록·통신판매업 신고 완료

## Consequences

### 출시 일정 단축

- v1 12주 → 8~9주로 단축 가능 (Week 9~10 결제, Week 11~12 베타·런칭 → 결제 제외 후 베타·런칭만)
- 사업자등록·약관 검토 v1 출시 차단 항목에서 제거

### Spec 영향

- F-PAY-01/03/04, F-CRD-01/04 P0 → **v2** (P3 보류)
- F-CRD-03 (크레딧 차감)도 v2 (충전 없으니 차감도 의미 없음)
- F-ADMIN-03 (결제·환불 모니터링)도 v2

### 정책 영향

- /policies/refund 페이지는 그대로 유지하되 "베타 기간 무료 운영 — 결제 활성 시 적용" 안내 추가
- /policies/terms §7 (Boost) 도 동일 처리
- /policies/credits §2 적립 정책은 유지, 사용 §3 는 v2 안내

### 베타 사용자 약속

- 사전 등록자에게 "베타 기간 무료" 명시
- v2 결제 활성 시 적립 크레딧 그대로 사용 가능 (소멸 X)

### 변호사 검토 일정

- v1 출시 전: 비활성 (약관·개인정보 자체 초안 v0.1 그대로)
- v2 진입 시: 검토 → v1.0 시행

### 관찰 지표 인프라 (선결 조건)

- PostHog 이벤트 트래킹 (signup, app_register, match_opt_in, checkin, complete)
- 일/주/월 리포트 자동 생성 (v2 진입 판정용)
- 본 ADR 후 별도 작업
