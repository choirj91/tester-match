# ADR-0003: 크레딧 모델 — 1원 = 1 크레딧

- **상태**: Accepted (2026-05-04 잠금)
- **결정자**: nakjun
- **관련**: 01-source/spec/06_erd.md §2.6, 01-source/spec/07_policies_draft.md §4

---

## Context

크레딧이 무엇을 표상하는지(노동 단위 vs 화폐 단위)는 회계·환급·정책·외부 전환·약관 모두에 영향. 잠금 필요.

### 후보

| 안 | 단위 | 매칭 비용 | 장점 | 단점 |
|---|---|---|---|---|
| A. 노동 단위 | 1 크레딧 = 14일 1매칭 완주 | 12 크레딧/12명 | 직관적, 노동 가치 명확 | KRW 환산 모호, 외부 포인트 전환 어려움, 부분 환불 복잡 |
| B. **화폐 단위 (KRW 1:1)** | 1 크레딧 = 1원 | 12,000 크레딧/12명 (= 12,000원) | 회계 단순, 부분 환불 정밀, 외부 전환(v3) 호환 | 노동 가치 시각화는 별도 UI 필요 |
| C. 추상 단위 (포인트) | 1 포인트 = ? | ? | 자유로운 가격 정책 | 사용자 혼란, 약관 복잡 |

### 적립 비율 후보 (B 채택 가정)

| 안 | 테스터 / 플랫폼 | 비고 |
|---|---|---|
| 70/30 | 700 / 300 | 플랫폼 마진 큼, 테스터 동기 약함 |
| **80/20** | **800 / 200** | 균형, 경쟁사 평균 |
| 90/10 | 900 / 100 | 테스터 매력도 ↑, 플랫폼 수익 모델 약함 |

---

## Decision

**B안 + 80/20 적립**.

- **단위**: 1 크레딧 = 1 KRW (1:1 고정)
- **매칭 1건 비용**: 1,000 크레딧 (= 1,000원)
- **테스터 적립**: 14일 완주 시 800 크레딧 (= 800원)
- **플랫폼 수수료**: 200 크레딧 (= 200원, 20%)
- **12명 모집 비용**: 12,000 크레딧 (= 12,000원)
- **충전 단위**: 1,000 / 5,000 / 10,000 / 30,000 / 50,000원 + 직접 입력
- **만료**: v1 무기한, v2부터 발급일 12개월 후 소멸 (사전 30일 알림)

### Boost는 별도 가격
- Boost 24h SLA: 49,000원 (매칭비 12,000원 + Boost 프리미엄 37,000원)
- Boost 48h SLA: 29,000원 (매칭비 12,000원 + Boost 프리미엄 17,000원)
- 테스터 적립은 Boost 매칭에서도 동일하게 800원/매칭 (총 9,600원/12명)

---

## Why

1. **회계 단순**: KRW 1:1이면 부가세·환불·세무 신고 자동 일치. 노동 단위는 환산 룰을 별도로 만들고 유지 필요.
2. **부분 환불 정밀**: 충전 후 7일 이내 환불 정책(07_policies §3.1)이 미사용 비율 정확히 계산 가능.
3. **외부 전환(v3) 호환**: 네이버페이/카카오페이는 KRW 기반 → 1:1이면 전환 비율 정의 단순 (1,000 크레딧 = 800 외부 포인트, 수수료 20%).
4. **사용자 신뢰**: "내가 800원 적립" > "내가 1 크레딧 적립" → 가치 인식 명확, 광고에 활용 가능.
5. **80/20 균형**: 경쟁사(LaunchPad, BETAFLOW) 분석 결과 70~85% 사이가 일반적. 80%는 테스터 매력도와 플랫폼 마진 균형점.

---

## Consequences

### 긍정
- credits_ledger 테이블의 amount 컬럼이 KRW로 직접 매핑 → 매출 신고 쿼리 단순
- 환불 정책(07 §3.1)이 PG 수수료 차감 후 비례 환불을 정확히 계산
- v3 외부 전환 활성화 시 "1,000 크레딧 = 800원 = 800 네이버포인트" 식으로 일관 안내

### 부정
- **선불전자지급수단 분류 위험**: 한국 전자금융거래법상 "1원 = 1 크레딧" + "외부 전환 가능"은 발행업 등록 대상 가능성 있음 → v3 외부 전환 활성화 전 금감원 사전 자문 필수 (Open Question §15.2)
- **AML 의무**: 양도/거래 활성화 시 자금세탁방지법 적용 가능 → v3까지 양도 비활성
- 테스터 활성도 인센티브가 약할 가능성 → Trust Score + 급구 매칭 800 → 1,600 크레딧(2배)로 보완

### 결정 갱신 트리거
- 금감원 자문 결과 "선불전자지급수단" 해당 → 외부 전환 정책 재설계 (수수료 비율 또는 전환 한도 조정)
- 테스터 14일 유지율 < 60% → 적립 비율 80/20 → 85/15 등 조정 검토
